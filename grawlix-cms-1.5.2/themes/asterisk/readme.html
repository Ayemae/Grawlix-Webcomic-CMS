Asterisk: Advanced layout for modern webcomics

Asterisk is a theme for the Grawlix CMS. It’s a collection of code and graphics you can use to quickly build a webcomic site design.

In addition to having tons of notes and instructions for you, Asterisk uses Flexbox to handle its layout. Flexbox is a relatively new CSS technique that makes arranging items much easier than with old-school floats & clears. If your readers use modern web browsers, then Asterisk is great. But if you think your readers are old-fashioned, then you might want to use Indotherm or Stick Figure.

Who should use this theme

We wrote Asterisk with forward-thinking webcomic artists in mind. If you’re looking for sophisticated, easy-to-apply layout, Asterisk is for you. You’ll need to change your mindset away from old CSS floats and clears, but it’s worth the effort in the long run.

What’s in the box

Asterisk is organized into three “page” PHP scripts, a dozen snippet files, five pattern files, and five CSS files. Among the CSS files are four “tones” which give the site different looks, although they use the same layout provided by the theme.css file. There are also 20 graphics, and you’ve already found the readme.

All of this is up for grabs. You can edit it to fit your needs, or just study the code as a model for your own theme.

What’s what

  • A theme is a collection of files that gives a webcomic site its unique look.
  • The themes folder is a special directory in the Grawlix CMS that contains themes. It resides at the top level of your webcomic site.
  • Theme.css is a CSS file that defines the site’s general layout.
  • A tone is a CSS file that modifies theme.css, usually by adding colors, typography, and graphics.
  • Selector: part of a CSS block that indicates what the block will affect. For example, .page__secondary { color: blue; } will make text blue inside of HTML that is classified as “page__secondary”.
  • Page.(something).php: Scripts that control the three different kinds of pages in a webcomic site: comic, static, and archive. Page scripts are the backbone of a theme. They use snippets to build out entire web pages.
  • Snippet.(something).php: One reusable piece of a larger web page. For example, snippet.menu-main.php contains just the site’s menu.
  • Pattern.(something).php: HTML that displays a static page block. Most static pages are made of discrete “bits” — like individual characters in a cast page. Patterns control how that information is structured.

Installing

  1. FTP: Upload the “asterisk” folder to the /themes folder in your Grawlix-based webcomic site.
  2. Install: Go to the “themes” section of your admin panel and click the Asterisk thumbnail to install it.
  3. Activate: Click the theme again. It should report that several tones are ready to go.
  4. Select: Back in the themes panel, choose the theme and tone from the drop-down list.

Customizing

HTML mixed in PHP can intimidate anyone learning the basics, so we wanted to lower the learning curve. You can change most of the site’s look and feel with just CSS and images. Check out the files theme.css and tone.zero-annotated.css to get oriented.

Change the banner

  1. Upload your banner graphic to the folder /theme/asterisk/images/custom.
  2. Depending on which tone you’ve selected in the admin panel, look in the files tone.mars.css or tone.fab.css (lines 76 and 31, respectively) for the selector .page__header h1 a. For its banner, the tone tone.grey.css uses text instead of an image.
  3. Change the background path and filename to your graphic. For example, if you named your banner my-banner.png, then change the code to background: url(images/custom/my-banner.png);.
Tech note: There’s nothing magical about the /themes/asterisk/images/custom folder. It’s there to help keep files organized, but you can put your graphics anywhere you feel is appropriate.

Change the background

  1. Upload your background graphic to the folder /themes/asterisk/images/custom.
  2. Depending on which tone you’ve selected in the admin panel, look in tone.mars.css or tone.verdant.css (line 20 in both) for the selector html,body.
  3. Change the background path and filename of that CSS block to your graphic. For example, if you named your banner my-background.jpg, then change the code to background: url(images/custom/my-background.png);.

Change the page width

Several items in CSS control the page width, but the main width property is in the .page__comic selector, around line 123 of theme.css. Also look for .page__secondary in the tone files.

Change Mars’ navigation graphics

To show how tones can work, the Mars tone replaces its text back/next links with graphics. You can change them like this:

  1. Upload your navigation graphics to /themes/asterisk/images/custom.
  2. Look around line 134 of tone.mars.css for the .navlink selector. Change its width and height properties to fit your graphics’ dimensions. (If your graphics are different sizes, then delete the width and height lines in the .navlink block.)
  3. Look for the .first, .prev, .next, and .last selectors in tone.mars.css (around lines 146–157). Change the paths of each background to the appropriate graphic. For example, if you named your “first” graphic beginning.gif, then the .first background property should read background: url(images/custom/beginning.gif);.
  4. If your back/next graphics are different sizes, then enter their width and height properties in the appropriate block, like .first and .next.
