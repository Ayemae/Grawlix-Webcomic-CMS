Welcome to v1.5

Versions 1.3 and 1.4 made major improvements to static pages — and upgrading from v1.2 means more than copying a few files.

What’s different

Version 1.3 and later let you rearrange content blocks in static pages and edit blocks’ layout HTML/CSS.

What to do

Upgrading isn’t as hard as it looks. Mostly you’ll upload files and copy/paste a bit of CSS.

First, back up your site’s files and, if possible, your site’s MySQL database. Just in case. And then:

  1. Upload the new _system and _admin folders into your website, replacing the originals.
  2. Upload the “pattern” files into your theme’s folder.
  3. Copy the CSS code from “add-to-theme.css” into your theme’s “theme.css” file.
  4. Log in to your admin panel.
  5. An info message will provide an “update” link. Click it once.
  6. Let the script do its thing.
  7. Delete the file named _upgrade-to-1.3.php in your _admin folder.
  8. Celebrate success.

New feature: Patterns

Pattern files are code snippets that give you control over static pages’ HTML and CSS. Want a wide text area with narrow images? You got it. Want larger headings or colorful text? Can do.

Like mad libs, each pattern is a file with blanks that are labeled with {curly brackets}. Content blocks fill in the blanks automatically with content you enter in the admin panel. The options in v1.3 and later are:

  • {id}
  • {title}
  • {image}
  • {content}
  • {link}

Patterns’ names reflect what they do — out of the box. You can edit any of them, so “image-right” doesn’t have to do what it implies. It’s just a label.

Do you have to edit the HTML/CSS? No. Out of the box it’s fine. But you can, and that’s the point.

Getting geeky

Here’s the stuff you don’t need to worry about. But if you’re curious …

Version 1.0 – 1.2 formatted static pages’ content in XML, and stored that XML in MySQL. Version 1.3 and later store content in a new MySQL table without XML. It’s easier for PHP to read and lets us rearrange content with relative ease. The field that used to hold XML now holds pattern-layout information.

Versions 1.4 and newer are smart enough to keep displaying pages that haven’t been converted from XML to MySQL — but the admin panel will insist that you upgrade before you can work with static pages again.

Happy publishing!
