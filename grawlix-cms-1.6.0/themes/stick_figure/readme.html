Stick Figure: a bare-bones theme to build upon

Stick Figure is a set of files that control the look of sites based on the Grawlix CMS. We designed it to be dismantled, rearranged and customized for any comic website.

See, we don’t want people to say “I can tell that’s a Grawlix site” at a glance. Instead, we want you to surprise the world with your creativity. Here we’ll explore how you can transform this starter theme into something uniquely yours.

Quick start

Many artists want to change a few things first:

  • The site’s banner
  • The site’s background
  • Link and text colors
  • Site width

The site banner

  1. Upload your banner image to /themes/stick_figure (the same folder in which you found this file).
  2. Open the file snippet.header.php.
  3. Replace the image src on line 26 with the path to your image. Don’t forget to use a sensible alt attribute.

The site’s background

There are two ways to change a site’s background color and image. Start by changing the parameters on lines 49–51 in theme.css. Then open the tone files (e.g. “tone.desert.css”) to make more specific changes — see themes and tones below.

Link and text colors

They’re in theme.css, right around lines 55–58. Tone files may override these colors, however — see Themes and Tones below.

Site width

Not everyone has the same screen as you, or views your site with their browser open as wide as possible. Comics that are, say, 1200 pixels wide must shrink to fit screens that display less than 1200 pixels across. So two factors influence your site’s layout: your comic image’s size and the reader’s browser size.

Found on lines 133–151 (for the comic image) and lines 97–114 (for the site), Stick Figure’s media queries control page and image widths on differently-sized browsers. In plain English they say, “if the screen is up to X pixels wide, then make the element no more than Y pixels wide.”

Media queries are not required. If you want to blast the theme.css file back to zero and write your own, no one will miss media queries unless your comic is wider than their browser.

But we strongly recommend you keep the media queries in place. The web is increasingly a mobile place, and if you want to stay modern, remember that not everyone has a glorious 21" full-color screen.

Best practice

  • Change the max-width of #comic figure on lines 134–151 of theme.css to match the images you upload.
  • Always make .container, main on lines 97–114 equal to or wider than your comic image.

Themes and tones

Stick Figure’s theme.css file controls the site’s general look: its layout, typography and broad colors. Tone files, whose names begin with “tone” and end with “css”, modify that look. For example, the theme.css file might say “make links blue,” but a tone file says, “no, make them orange.”

The theme.css file applies to your whole site. Tone files, though, you can apply to individual chapters or static pages. That means you could, say, make your FAQ and Cast pages greyscale, but your comic in full color. Or if your story changes from winter to summer in two chapters, each can have its own motif.

Assign themes and tones to your site in “Themes” of your admin panel.

PHP functions

Don’t panic, we kept the arcane stuff to a minimum. But you’ll come to recognize a few essentials.

Snippet()

Snippets are PHP files whose names (surprise) begin with “snippet” and end with “php”. The snippet function grabs such a file and sticks it into place. For example, snippet('footer') adds the file snippet.footer.php. Check out the docs for details.

Show()

“Show” functions grab bits of data from the admin panel. For example, show('artist_name') inserts the name you provide in Settings of your admin panel. See the docs for more info.

PHP scripts

So, what’s in the box? Every Grawlix theme comes with an assortment of PHP scripts that generate web pages that the reader ultimately sees. Each script is part of a whole page.

Page.archive

Displays links to your comic pages, in order. Control its appearance (what the links’ text is) in the admin panel.

Page.comic

Displays comic images — the “comic pages” themselves — as well as the blog post snippet, transcript snippet, sharing snippet, etc.

Page.static

Displays content for non-comic pages like the FAQ and “About” page.

Snippet.archive-nav

Comic navigation links that includes one to the archives page.

Snippet.comic-nav

The first/previous/next/latest links in your list of comic pages.

Snippet.comic-nav-random

More comic navigation links … with an extra link to a certain comic page. Which page? Who knows? It’s different every time, giving readers a surprise at every page load.

Snippet.comments

Contains code that enables readers to leave their thoughts on your comic pages — Disqus, etc. Tinker with this only if you get updated code from your commenting service of choice.

Snippet.follow

Contains links to other places on the ’net people can find you, like RSS, Facebook, Pinterest and Twitter. Best left alone unless you know a little PHP.

Snippet.footer

The end of every web page on your site. This snippet includes the copyright statement and publish frequency, both of which you change in the admin panel or custom-write by simply replacing all the code in line 5.

Snippet.header

Begins each web page with info about the HTML, favicons, and — by default — site banner, title and site-wide navigation menu.

Snippet.main-menu

Displays a group of links for the site-wide navigation or menu. We left this file deliberately simple in case you want to make it more complex with, for example, hard-coded links.

Snippet.share

This complicated-looking script gives readers the chance to tell other people about your comic pages via social media. You can control which appears in the site via your admin panel’s sharing options.

Snippet.twitterstream

Puts a Twitter link into your comic. Best left alone unless you know PHP or have your own Twitter link code; enter your Twitter info in the admin panel to make this snippet work.

Snippet.transcript

Shows a comic page’s written dialog, events, setting, etc to the reader.

Pattern.(something).php

Pattern files are bits of HTML that structure static pages’ content blocks. You can create your own pattern files or edit the included ones to fit your needs.

Various HTML voodoo

Stick Figure is based on HTML5, and includes smart (yet optional) semantics to make your site searchable and readable by a variety of devices and services.

Article

“Articles” are general containers that can stand alone. Think of them as articles of clothing: individual things that work together for a successful look.

Figure

“Figure” contains an image and related information. We use <figure> as the principle container for your comic images.

Itemx

“Itemtype” and “itemprop” help search engines (coughgoogle) figure out what different elements are all about. They’re helpful in SEO for search engine results pages.

Role

“Role” also helps search engines like Google and screen readers like Jaws make sense of raw HTML.

Learn more about elements like <figure>, <main> and <header> at HTML5 Doctor or with our ebook.
